18// Idea: schedule command takes in a function pointer. Then, a new command scheduler
19// structure should be created, containing a serial number, the function pointer,
20// and an Status containing the completion status. When a command is scheduled to run with
21// the schedule_command() function, the unique serial number is returned. When a job is scheduled and
22// run, it is popped off the queue, and the status variable associated with the structure is
23// updated. Finally, the structure is pushed onto another vector containing completed jobs.
24// the number of completed jobs that are stored is limited to a predefined number.
25// Finally, there should be a function that takes argument for a serial number associated
26// with a command, and returns the completion status.
